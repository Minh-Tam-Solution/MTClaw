## Executive Summary

MTClaw transforms MTS-OpenClaw from a chat tool into a **governance backbone for AI-first transformation**. Built on GoClaw runtime (Go, PostgreSQL multi-tenant), it provides 3 governance rails and 16 role-aware AI personas (SOULs) for 10 MTS employees (Phase 1), expandable to 100-150 NQH employees (Phase 2).

## Gate Checklist

### 1. Problem Statement
- [x] **Who**: 10 MTS employees (Engineering, Sales, CS, Back Office)
- [x] **Current state**: MTS-OpenClaw = chat tool, no governance rails
- [x] **Desired state**: 3 Rails governance + 16 SOULs + Bflow AI integration
- [x] **Root cause**: Generic AI (chat.nhatquangholding.com) lacks business context, persistent memory, role-aware personas
- **Doc**: [problem-statement.md](problem-statement.md)

### 2. Business Case
- [x] **ROI**: ~393M VND/year savings vs ~35M VND/year cost = payback < 1 month
- [x] **Strategic**: Governance backbone → OaaS foundation (Phase 3)
- [x] **Risk**: Go competency gap mitigated (ADR-001: AI Codex + CTO gate + 90-day eval)
- **Doc**: [business-case.md](business-case.md)

### 3. User Research (8 Interviews)

- [x] **Back Office** (n=2, small sample): Need contract templates, HR policy lookup
- [x] **Root cause consensus**: 8/8 — generic AI, no Bflow context, no persistent memory
- [x] **Adoption signal**: 6/8 Yes, 2/8 Maybe (Engineering accuracy concern = adoption gate)
