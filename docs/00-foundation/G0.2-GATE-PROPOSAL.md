### 6. Architecture (Design Phase Artifacts)

#### 6a. SOUL Loading Implementation Plan

- [x] **GoClaw system prompt analysis**: 15-section `BuildSystemPrompt()` fully documented
- [x] **3 injection points identified**: `agent_context_files` (SOUL.md), `ExtraPrompt` (governance), `ContextFiles` (bootstrap)
- [x] **Database seeding plan**: 16 agents + 48 context files (3 per SOUL)
- [x] **SOUL routing design**: Default resolution → delegation → handoff
- [x] **Governance extensions**: ExtraPrompt for rail-specific context
- **Doc**: [soul-loading-implementation-plan.md](../02-design/soul-loading-implementation-plan.md)

#### 6b. RLS Tenant Isolation Design

- [x] **Gap analysis**: No RLS policies in current GoClaw schema
- [x] **Design**: `owner_id` as tenant identifier + `SET LOCAL app.tenant_id` middleware
- [x] **RLS policies**: 8 tables with row-level security (agents, context_files, sessions, memory, traces, spans)
- [x] **Admin bypass**: `mtclaw_admin` role for migrations/dashboard
- [x] **Cost guardrails**: Monthly token limit + daily request limit + soft-throttle at 80%/100%
- [x] **Testing strategy**: Unit tests (cross-tenant blocked) + integration tests (delegation, cost)
- **Doc**: [rls-tenant-isolation-design.md](../02-design/rls-tenant-isolation-design.md)

#### 6c. /spec Command Design

- [x] **Architecture decision**: Skill-based approach (Git-versioned SKILL.md, hot-reloadable)
- [x] **Sprint 4 prototype**: `/spec` → PM SOUL → JSON spec → evidence attachment
- [x] **Sprint 7 full**: spec_id, BDD scenarios, risk scoring, evidence vault link
- [x] **SOUL routing**: All SOULs delegate to PM for spec generation
- [x] **Example interactions**: Vietnamese + English examples with expected output
- **Doc**: [spec-command-design.md](../02-design/spec-command-design.md)
